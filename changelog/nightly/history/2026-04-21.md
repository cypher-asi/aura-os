# Debug app debuts, run state survives reloads

- Date: `2026-04-21`
- Channel: `nightly`
- Version: `0.1.0-nightly.322.1`
- Release: https://github.com/cypher-asi/aura-os/releases/tag/v0.1.0-nightly.322.1

Today's nightly centers on making in-flight and past runs legible. A new Debug app and companion CLI turn every dev-loop run into an inspectable bundle, while a wave of rehydration work keeps the Run panel, sidekick tabs, and live indicators consistent across refreshes and WebSocket reconnects. A browser backend vendor fork also lands to start untangling a long-running CDP log flood.

## 10:52 PM — Completed task output rehydrates after reload

The Run panel now keeps structured task output available even after pruning or a full page refresh, and a small sidekick layout fix prevents stat values from wrapping.

- Expanding a completed task row now replays the full timeline, tool cards, and thinking steps from a new localStorage task-turn-cache, with a unified useTaskOutputView hook that walks live stream, cache, event store, and server hydration in order so rows no longer render empty after TTL pruning or a reload. (`657ef48`)
- Sidekick StatCard values like "13m 53s" now stay on a single line in the narrow column by shrinking the value to 15px and adding nowrap, while the mobile variant keeps its larger size. (`b2f4907`)

## 11:18 PM — chromiumoxide vendored to trace CDP WebSocket warnings

The browser backend's chromiumoxide dependency is now vendored at v0.9.1 so the team can surface the raw CDP payload behind a recurring "WS Invalid message" warning flood.

- Vendored chromiumoxide 0.9.1 under vendor/chromiumoxide and pinned the workspace via [patch.crates-io], with a single local edit that includes the raw JSON payload on the handler's InvalidMessage WARN so the specific drifting CDP method can be identified and patched in follow-ups. (`81728bf`)

## 11:42 PM — Live run state survives refresh across sidekick, Tasks, and stats

A cluster of fixes makes the automation loop's live state reliable after reload: the server now exposes active tasks over HTTP, the stats dashboard updates in real time, the sidekick remembers its tab, and the obsolete bottom Task Output panel is removed.

- After a page refresh, the Run panel row, Tasks-tree "live" dot, and left-nav agent spinner now rehydrate from a new active_tasks field on /loop/status, /loop/start, /loop/pause, /loop/stop, and /loop/resume, instead of waiting for a task_started WebSocket event that is never replayed. (`c16c726`)
- The Stats dashboard's Active card now uses the app's success green and updates optimistically from TaskStarted/TaskCompleted/TaskFailed events scoped to the current project, so in-progress, done, and failed counts track the automation loop without waiting for a chat stream to end. (`9408bf2`)
- The sidekick now remembers the last-active tab for main, agents, and process surfaces in localStorage, so refreshing no longer snaps users back to terminal, profile, or process defaults. (`8ec8861`)
- Removed the long-deprecated bottom TaskOutputPanel surface, including its resize handle, collapse state, and the stale aura-task-output-panel localStorage key, which is now cleaned up on load; the Sidekick drawer's Run and Terminal panes are the only remaining surface. (`c351ca8`)

## 12:28 AM — Debug app and aura-run-analyze CLI ship for dev-loop runs

Every dev-loop run is now captured to disk as a structured bundle, surfaced through a new Debug app and a companion Rust CLI with heuristics, while the Run button gets an instant-response fix and process Live Output also survives reloads.

- Dev-loop runs are now always written to a structured on-disk bundle (metadata.json, per-channel JSONL, summary.md), exposed via a new /api/debug/* surface and a Debug app with project/run navigation, a virtualized timeline, channel/type/text filters, a JSON inspector, and .zip export for analyzing LLM retries, blockers, and iteration cost. (`c18bc88`)
- Runs are now associated with the specs their tasks touched, with the Debug nav grouping Project → Spec → Runs and the run list filterable by spec; a new aura-loop-log-schema crate shares the on-disk format, and a new aura-run-analyze CLI (--latest/--run/--list, markdown or JSON) runs seven heuristics like repeated_blocker_path, high_retry_density, and token_hog_llm_call, exiting non-zero so CI can gate on warnings or errors. (`ccea501`)
- The Run preview's Live Output panel now rehydrates from a new process-node-turn-cache after reloads or WebSocket reconnects, with liveRunNodeId persisted per run and the heading switching to "Recent Live Output" once the run is no longer streaming, so the panel no longer flashes empty mid-run. (`3156ede`)
- Clicking Run now shows the active task instantly: the Run button and sidekick Run/Tasks spinners stay engaged continuously from click through the first task_started event, and the Run panel plus Tasks "live" dot seed from the active_tasks returned by /loop/start and /loop/resume, eliminating the idle flicker during the HTTP→WS handoff, including on re-runs. (`9270310`)
- Cargo is now configured workspace-wide with color=never and progress.when=never so agent and CI transports that Base64-wrap streams produce legible compiler diagnostics instead of ANSI-laden blobs; humans can still pass --color=always. (`de66693`)

## Highlights

- New Debug app and aura-run-analyze CLI for inspecting dev-loop runs
- Task and process run output now survives page reloads
- Sidekick remembers your last tab per app
- chromiumoxide vendored to diagnose CDP WebSocket warnings
