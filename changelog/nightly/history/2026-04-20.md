# Agent permissions get trustworthy, and desktop learns to update itself

- Date: `2026-04-20`
- Channel: `nightly`
- Version: `0.1.0-nightly.312.1`
- Release: https://github.com/cypher-asi/aura-os/releases/tag/v0.1.0-nightly.312.1

A heavy day centered on making agent capabilities actually stick — a unified tool registry, end-to-end permission reconciliation, and autosaving toggles — alongside a new in-app updater control for desktop, faster chat loads, and tougher CI for Android and packaging.

## 9:29 PM — Unified agent tool registry and skill-delete safety net

A late-night push rebuilt how agent capabilities map to live tools, closed several permission-propagation bugs, sped up chat history loads, and added an in-app updater to Settings.

- Replaced the old tier-1/tier-2 + CEO-preset branching with a single capability-aware tool registry where each tool declares a Surface (Always/OnDemand); CEO and regular agents now flow through the same filter, restoring missing tools like list_agents, send_to_agent, and list_agent_instances in CEO sessions. (`2a7ac33`)
- Capability changes now propagate without a server restart: update_agent invalidates the direct chat session, the dispatcher permissions cache, and every instance-scoped session owned by the agent, and project-instance chats re-read fresh parent capabilities each cold start. (`2a7ac33`, `76b34a6`, `9f9784b`)
- Deleting a user-authored skill now refuses with a 409 and an inline blocker list when it's still installed on other agents, and the Available catalog hides entries whose SKILL.md has been removed — no more orphaned installs or zombie catalog rows. (`73b0ecb`)
- Added an Update status block under the Settings Version row — check, download, install, and failure states on desktop, with a graceful "managed by the server" fallback on web — and promoted it to a shared UpdateControl that also appears in Team Settings → General. (`267eb96`, `6b6910c`, `823c16d`)
- Opening a chat with lots of past sessions no longer reads every historical event: per-session event loads now fan out in parallel batches of four and stop as soon as the requested window is filled. (`1e7986c`)

## 10:46 PM — Team Settings About group layout polish

Small visual cleanup to the new update control's placement in Team Settings.

- Reordered the About group to Version / Commit / Built / Updates so the interactive upgrade control sits at the end of the static build metadata instead of splitting the Version row. (`d2bc9a8`)
- The "updates managed by the server" fallback now wraps and right-aligns as two balanced lines instead of overflowing the row on unsupported browsers. (`969a9a9`)

## 10:54 PM — Instant chat paint and autosaving permission toggles

Chat first-paint got dramatically faster, permission edits now persist automatically, and a long-standing bug where saved toggles silently reverted after restart is finally closed.

- Chat history now persists to IndexedDB and hydrates on mount, so reloads paint the last-known conversation immediately while the network revalidates in the background; private-browsing falls back to localStorage. (`588bdf3`)
- Cold-boot waterfalls on /agents shrank: AgentChatView is now eagerly imported, AgentIndexRedirect redirects synchronously off the cached last-agent id, project-agent discovery is cached server-side with a 30s TTL, and sidebar preview prefetches wait for the foreground chat and run during idle time at reduced concurrency. (`6f149de`, `3cfc04d`, `2b05176`)
- The Permissions tab dropped the Save/Discard bar: edits autosave ~350ms after the last change with debounced PUTs, a status pill shows Saving/Saved/Save failed with a Retry action, and pending drafts flush on unmount so nothing is silently lost. (`7f67bdf`)
- Fixed the "toggles stick for a session then vanish on restart" regression: read paths now reconcile an empty permissions column from aura-network against the local shadow across list_agents, get_agent, hydrate, and installed_tools, so saved capabilities survive app boot. (`5c5c565`)

## 11:23 PM — Android release preflight for missing signing secrets

The Android shipping pipeline now surfaces every missing secret up front instead of failing deep inside Fastlane.

- A new preflight script checks every required environment variable for the selected Fastlane lane in one pass and prints actionable guidance; wired into both android-mobile.yml (ship-android) and release-mobile-nightly.yml so signed-build and Play Store paths fail loudly and early. (`2cce560`)

## 10:57 AM — Desktop packaging retries on transient failures

Desktop release workflows now recover automatically from the most common flaky packaging errors instead of failing the whole build.

- Packaging now retries up to three times on transient macOS hdiutil "Resource busy" errors and Linux AppRun 429/5xx download failures, cleaning up stale DMG mounts between attempts via a shared cleanup-stale-macos-dmg.py script used across desktop-validate, release-nightly, and release-stable. (`66ff7e7`)

## Highlights

- Capability toggles now take effect immediately and survive restarts
- New in-app update control in Settings and Team Settings
- Chat opens faster via IndexedDB cache and parallel history reads
- CI fails fast on missing Android secrets and retries transient packaging failures
