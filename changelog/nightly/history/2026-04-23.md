# AURA 3D studio lands, GPT-5.5 arrives, and nightly release pipeline hardens

- Date: `2026-04-23`
- Channel: `nightly`
- Version: `0.1.0-nightly.359.1`
- Release: https://github.com/cypher-asi/aura-os/releases/tag/v0.1.0-nightly.359.1

A big day for Aura: a brand-new AURA 3D app went from scaffold to a polished image-to-3D studio with a Three.js viewer, artifact persistence, and project-scoped history. Alongside it, GPT-5.5 support landed in chat, the nightly release and changelog-media pipelines got meaningfully more resilient, and a lingering SSE proxy bug that silently dropped generation events was fixed.

## 9:17 AM — Nightly release pruning and changelog history get failure-tolerant

The nightly release workflow and changelog media pipeline were restructured around retry-safe scripts and a new recovery workflow so transient GitHub failures no longer break publishing.

- Extracted nightly asset pruning into a dedicated retry-aware script and added a sync workflow that reconciles changelog media history from gh-pages after each publish run. (`a7eb25a`, `ac61ac3`)
- Added a gh-pages republish recovery workflow backed by a retrying commit script, so a failed changelog push can be replayed without hand intervention. (`ca9eaa8`)
- Reworked changelog media publishing to sync into dated history entries and improved framing and readability of the generated proof cards. (`d81834c`, `2217600`, `43ac905`)

## 5:11 PM — SSE generation proxy no longer drops data-only frames

A quiet failure mode in the generation proxy was fixed: upstream events without an explicit event line are now parsed correctly instead of silently discarded.

- The aura-os-server generation handler now extracts the event type from the JSON data field when the upstream router sends data-only SSE frames, unblocking generation streams that previously stalled. (`e0d60fd`)

## 5:11 PM — AURA 3D ships artifact persistence and goes on by default

- Hooked the frontend store up to the artifacts API so selecting a project loads its saved images and models, strips style-lock prompt suffixes for display, and preserves image-to-model lineage via sourceImageId/parentId. (`f3dc0ae`, `e204d4c`)
