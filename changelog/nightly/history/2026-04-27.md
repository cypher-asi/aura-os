# Token stats and per-project cost attribution restored

- Date: `2026-04-27`
- Channel: `nightly`
- Version: `0.1.0-nightly.409.1`
- Release: https://github.com/cypher-asi/aura-os/releases/tag/v0.1.0-nightly.409.1

Today's nightly closes a long-standing gap that left the dashboard's Tokens stat stuck at zero and quietly excluded several LLM round-trips from per-project cost rollups. The fix re-wires task-completion persistence and stamps the missing attribution headers on summary and tool-session calls.

## 9:09 AM — Task token usage now persists on completion and failure

A dormant persistence path is finally wired up so token counts and task output stop being silently dropped when tasks finish, and previously unattributed router calls now carry the headers needed for per-project cost aggregation.

- Task completion and failure handlers now drain the in-memory task output cache into aura-storage, so accumulated input/output tokens, files-changed, and live output actually land on the task record instead of being discarded — the root cause behind the dashboard's Tokens stat showing 0. (`c4435e0`)
- Session summary generation now stamps x-aura-session-id, x-aura-project-id, and x-aura-agent-id on its router call, threading project and agent IDs from the URL so the resulting token usage is no longer recorded with a null project_id and dropped from per-project cost rollups. (`c4435e0`)
- Tool sessions used by spec-generate and task-extract now resolve and attach the project's aura_org_id, and aura-router has been loosened to accept project_id alone, so utility tool calls without a live storage session still get attributed to the right project. (`c4435e0`)

## Highlights

- Dashboard Tokens stat now reflects real task usage
- Session summaries and tool sessions count toward per-project cost
